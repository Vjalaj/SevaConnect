## 🌟 What is SevaConnect?

**SevaConnect** is a comprehensive, modern donation platform that revolutionizes charitable giving by making it simple, secure, and meaningful. Built with cutting-edge technology and designed with love, it serves as a bridge between generous donors and various charitable causes, ensuring every contribution makes a real impact.

### 🎯 Our Mission
To democratize charitable giving by creating an accessible, transparent, and impactful platform where generosity meets technology, enabling everyone to contribute to causes they care about with confidence and ease.

### 🏆 Why Choose SevaConnect?
- **🔒 Bank-Level Security**: Google OAuth, encrypted payments, and secure data handling
- **💳 Multiple Payment Options**: Flexibility with 3 different payment methods
- **📱 Mobile-First Design**: Seamless experience across all devices
- **⚡ Lightning Fast**: Optimized performance with Next.js 15
- **🎨 Beautiful UI/UX**: Thoughtfully designed with accessibility in mind
- **🔧 Admin-Friendly**: Powerful yet simple content management system

## 🚀 Complete Setup Guide

### 📋 Prerequisites Checklist

Before starting, ensure you have:
- ✅ **Node.js 18+** installed ([Download here](https://nodejs.org/))
- ✅ **Git** for version control ([Download here](https://git-scm.com/))
- ✅ **Google Account** for admin authentication
- ✅ **Text Editor** (VS Code recommended)
- ✅ **Terminal/Command Prompt** access

### 🔧 Installation Process

#### **Step 1: Clone and Setup**
```bash
# Clone the repository
git clone https://github.com/your-username/sevaconnect.git
cd sevaconnect

# Install dependencies (this may take 2-3 minutes)
npm install

# Copy environment template
cp .env.example .env.local
```

#### **Step 2: Environment Configuration**
Open `.env.local` in your text editor and configure the following:

---

### 📧 **Email Configuration (Required)**

```env
ADMIN_EMAIL=your_admin_email@gmail.com
NEXT_PUBLIC_ADMIN_EMAIL=your_admin_email@gmail.com
RECIPIENT_EMAIL=your_organization_email@gmail.com
NEXT_PUBLIC_RECIPIENT_EMAIL=your_organization_email@gmail.com
```

**📝 Detailed Explanation:**
- **ADMIN_EMAIL**: The email address that can access the admin panel (server-side)
- **NEXT_PUBLIC_ADMIN_EMAIL**: Same email for client-side authentication checks
- **RECIPIENT_EMAIL**: Email that receives donation notifications (server-side)
- **NEXT_PUBLIC_RECIPIENT_EMAIL**: Same email for client-side form submissions

**🎯 Pro Tips:**
- Use the same Gmail account for admin access that you'll use to log in
- The recipient email can be different (your organization's main email)
- These emails will receive all donation notifications via FormSubmit

---

### 🔐 **Google OAuth Setup (Required for Admin Access)**

```env
GOOGLE_CLIENT_ID=your_google_client_id
GOOGLE_CLIENT_SECRET=your_google_client_secret
```

**📋 Step-by-Step Google Cloud Console Setup:**

#### **1. Create Google Cloud Project**
1. 🌐 Visit [Google Cloud Console](https://console.cloud.google.com/)
2. 🔑 Sign in with your Google account
3. 📁 Click "Select a project" → "New Project"
4. ✏️ Project name: "SevaConnect" (or your preferred name)
5. 🏢 Organization: Leave as default or select your organization
6. ✅ Click "Create" and wait for project creation

#### **2. Enable Required APIs**
1. 📚 Go to "APIs & Services" → "Library"
2. 🔍 Search for "Google+ API" or "Google Identity"
3. 📋 Click on "Google+ API"
4. ✅ Click "Enable" button
5. ⏳ Wait for API to be enabled (usually instant)

#### **3. Configure OAuth Consent Screen**
1. 🔧 Go to "APIs & Services" → "OAuth consent screen"
2. 👤 Choose "External" user type
3. 📝 Fill required fields:
   - **App name**: SevaConnect
   - **User support email**: Your email
   - **Developer contact**: Your email
4. 💾 Save and continue through all steps

#### **4. Create OAuth Credentials**
1. 🔑 Go to "APIs & Services" → "Credentials"
2. ➕ Click "Create Credentials" → "OAuth 2.0 Client IDs"
3. 🌐 Application type: "Web application"
4. ✏️ Name: "SevaConnect Auth"
5. 🔗 **Authorized JavaScript origins**:
   - `http://localhost:9002` (for development)
   - `https://yourdomain.com` (for production)
6. 🔄 **Authorized redirect URIs**:
   - `http://localhost:9002/api/auth/callback/google` (development)
   - `https://yourdomain.com/api/auth/callback/google` (production)
7. ✅ Click "Create"

#### **5. Copy Credentials**
1. 📋 Copy "Client ID" → paste as `GOOGLE_CLIENT_ID`
2. 🔐 Copy "Client Secret" → paste as `GOOGLE_CLIENT_SECRET`
3. 💾 Save your `.env.local` file

**⚠️ Important Security Notes:**
- Never commit your `.env.local` file to version control
- Keep your Client Secret confidential
- Regularly rotate your credentials for security

### 🔒 **NextAuth Configuration (Required)**

```env
NEXTAUTH_SECRET=your_nextauth_secret_key
NEXTAUTH_URL=http://localhost:9002
```

**🔑 Generate Secure Secret:**

#### **Method 1: Online Generator (Recommended)**
1. 🌐 Visit [Generate Secret](https://generate-secret.vercel.app/32)
2. 📋 Copy the generated 32-character string
3. 📝 Paste as `NEXTAUTH_SECRET`

#### **Method 2: Command Line**
```bash
# On macOS/Linux
openssl rand -base64 32

# On Windows (PowerShell)
[System.Web.Security.Membership]::GeneratePassword(32, 0)
```

#### **Method 3: Node.js**
```javascript
// Run in Node.js console
require('crypto').randomBytes(32).toString('base64')
```

**🌐 URL Configuration:**
- **Development**: `http://localhost:9002`
- **Production**: `https://yourdomain.com`

### 💳 **Razorpay Payment Gateway Setup**

```env
RAZORPAY_KEY_ID=your_razorpay_key_id
RAZORPAY_KEY_SECRET=your_razorpay_key_secret
NEXT_PUBLIC_RAZORPAY_KEY_ID=your_razorpay_key_id
```

**📋 Complete Razorpay Setup Process:**

#### **1. Account Creation**
1. 🌐 Visit [Razorpay.com](https://razorpay.com/)
2. 📝 Click "Sign Up" → Fill business details
3. 📧 Verify email address
4. 📱 Verify phone number with OTP

#### **2. Business Verification (KYC)**
1. 🏢 Complete business profile:
   - Business type (Individual/Company/Partnership)
   - Business category (NGO/Charity recommended)
   - Annual turnover
   - Business address
2. 📄 Upload required documents:
   - **For Individual**: PAN Card, Aadhaar Card, Bank Statement
   - **For Company**: Certificate of Incorporation, PAN Card, Bank Statement
   - **For NGO**: 12A/80G Certificate, Registration Certificate
3. ⏳ Wait for approval (24-48 hours typically)

#### **3. Get API Keys**
1. 🔑 Go to Dashboard → Settings → API Keys
2. 🧪 **For Testing**: Click "Generate Test Key"
3. 🚀 **For Production**: Click "Generate Live Key" (after KYC approval)
4. 📋 Copy credentials:
   - **Key ID**: Starts with `rzp_test_` or `rzp_live_`
   - **Key Secret**: Keep this confidential

#### **4. Webhook Configuration (Optional)**
1. 🔗 Go to Settings → Webhooks
2. ➕ Add webhook URL: `https://yourdomain.com/api/webhooks/razorpay`
3. ✅ Select events: `payment.captured`, `payment.failed`

**💡 Pro Tips:**
- Start with test keys for development
- Switch to live keys only after thorough testing
- Monitor transactions in Razorpay dashboard
- Set up webhooks for automatic payment verification

### 💰 **Instamojo Payment Gateway Setup**

```env
INSTAMOJO_API_KEY=your_instamojo_api_key
INSTAMOJO_AUTH_TOKEN=your_instamojo_auth_token
INSTAMOJO_ENDPOINT=https://test.instamojo.com/api/1.1/
```

**📋 Complete Instamojo Setup Process:**

#### **1. Account Registration**
1. 🌐 Visit [Instamojo.com](https://www.instamojo.com/)
2. 📝 Click "Sign Up" → Choose account type
3. 📧 Fill registration details and verify email
4. 📱 Complete phone verification

#### **2. Profile Completion**
1. 🏢 Business Information:
   - Business name and type
   - Category (select "Non-profit" if applicable)
   - Business address and contact details
2. 🏦 Bank Account Details:
   - Account holder name
   - Account number and IFSC code
   - Upload cancelled cheque or bank statement
3. 📄 Document Upload:
   - PAN Card (mandatory)
   - Aadhaar Card or Voter ID
   - Business registration documents (if applicable)

#### **3. Account Verification**
1. ⏳ Wait for document verification (1-3 business days)
2. 📧 Check email for verification status
3. 📞 May receive verification call from Instamojo team

#### **4. API Credentials**
1. 🔑 Go to Dashboard → Settings → API & Plugins
2. 📋 Find "API Details" section
3. 📝 Copy credentials:
   - **API Key**: Long alphanumeric string
   - **Auth Token**: Another long alphanumeric string

#### **5. Endpoint Configuration**
- **Testing**: `https://test.instamojo.com/api/1.1/`
- **Production**: `https://www.instamojo.com/api/1.1/`

**⚠️ Important Notes:**
- Instamojo charges 2% + GST per transaction
- Minimum transaction amount: ₹9
- Settlement time: T+2 working days
- Supports UPI, cards, wallets, and net banking

## 🔐 Advanced Security Implementation

### 🛡️ **Authentication Security**

#### **Google OAuth Flow**
```typescript
// Secure OAuth configuration
const authOptions = {
  providers: [
    GoogleProvider({
      clientId: process.env.GOOGLE_CLIENT_ID!,
      clientSecret: process.env.GOOGLE_CLIENT_SECRET!,
    }),
  ],
  callbacks: {
    async signIn({ user }) {
      // Restrict access to specific email
      return user.email === process.env.ADMIN_EMAIL;
    },
    async session({ session, token }) {
      // Add additional security checks
      return session;
    },
  },
  pages: {
    signIn: '/admin',
    error: '/admin',
  },
};
```

#### **Session Management**
- **JWT Tokens**: Secure, stateless authentication
- **Session Expiry**: Automatic logout after inactivity
- **CSRF Protection**: Built-in Next.js CSRF protection
- **Secure Cookies**: HttpOnly, Secure, SameSite cookies

- **Automatic Deployments**: Git-based deployments
- **Edge Functions**: Global performance
- **Zero Configuration**: Works out of the box
- **Free Tier**: Generous limits for small projects

#### **Step-by-Step Deployment**

**1. Prepare Your Repository**
```bash
# Ensure all changes are committed
git add .
git commit -m "feat: ready for deployment"
git push origin main
```

**2. Connect to Vercel**
1. Visit [vercel.com](https://vercel.com)
2. Sign up with GitHub account
3. Click "New Project"
4. Import your SevaConnect repository
5. Configure project settings

**3. Environment Variables Setup**
```bash
# Add all environment variables in Vercel dashboard
ADMIN_EMAIL=your_admin_email@gmail.com
NEXT_PUBLIC_ADMIN_EMAIL=your_admin_email@gmail.com
RECIPIENT_EMAIL=your_recipient_email@gmail.com
NEXT_PUBLIC_RECIPIENT_EMAIL=your_recipient_email@gmail.com

# Google OAuth (update redirect URLs)
GOOGLE_CLIENT_ID=your_google_client_id
GOOGLE_CLIENT_SECRET=your_google_client_secret

# NextAuth (generate new secret for production)
NEXTAUTH_SECRET=your_production_secret
NEXTAUTH_URL=https://your-domain.vercel.app

# Payment Gateways
RAZORPAY_KEY_ID=your_razorpay_key_id
RAZORPAY_KEY_SECRET=your_razorpay_key_secret
NEXT_PUBLIC_RAZORPAY_KEY_ID=your_razorpay_key_id

INSTAMOJO_API_KEY=your_instamojo_api_key
INSTAMOJO_AUTH_TOKEN=your_instamojo_auth_token
INSTAMOJO_ENDPOINT=https://www.instamojo.com/api/1.1/
```

**4. Update OAuth Settings**
- Go to Google Cloud Console
- Update authorized origins: `https://your-domain.vercel.app`
- Update redirect URIs: `https://your-domain.vercel.app/api/auth/callback/google`

**5. Deploy**
- Click "Deploy" in Vercel
- Wait for build completion (2-3 minutes)
- Visit your live site

1. **Purchase Domain**: From any registrar (Namecheap, GoDaddy, etc.)
2. **Add to Vercel**: Project Settings → Domains
3. **Configure DNS**: Point domain to Vercel
4. **SSL Certificate**: Automatic HTTPS setup
5. **Update Environment**: Change `NEXTAUTH_URL` to your domain
